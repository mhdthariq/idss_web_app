# IDSS Web App

**Intelligent Decision Support System for Accounts Receivable Late Payment Risk Prediction**

A modern web application (FastAPI backend + Next.js frontend) that serves dual-model predictions (XGBoost + MLP) for accounts receivable late payment risk assessment.

### Self-Contained Deployment

This project is **fully self-contained**. All required artifacts — `src/`, `models/`, `data/`, and `docs/` — are included directly inside `idss_web_app/`. There is **no dependency** on the parent `debt_prediction/` project at runtime.

The backend resolves all paths relative to `idss_web_app/` itself:

- `src/` is added to `sys.path` so `from src.features import CONSERVATIVE_FEATURES` works.
- `models/` contains all pickled model weights and pipeline maps.
- `data/processed/` contains CSVs, JSONs, and PNG figures served by the API.
- `docs/figures/` contains static thesis figures served by the API.

You can deploy this directory anywhere (Docker, cloud VM, etc.) without needing the original `debt_prediction/` project.

## Setup

### 1. Initialise the virtual environment

```bash
cd idss_web_app
uv sync
```

This creates a `.venv/` inside `idss_web_app/` with all required dependencies.

### 2. Verify artifacts

```bash
uv run python run.py --check-artifacts
```

This checks that all required model files exist and reports their status.

### 3. Start the backend server

**Development (with auto-reload):**

```bash
uv run python run.py --reload
```

**Or directly with uvicorn:**

```bash
uv run uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

**Production:**

```bash
uv run python run.py --workers 4
```

The API will be available at:

- **Swagger UI (interactive docs):** http://localhost:8000/docs
- **ReDoc:** http://localhost:8000/redoc
- **OpenAPI JSON:** http://localhost:8000/openapi.json
- **Health check:** http://localhost:8000/api/health

## Key Design Decisions

### 1. Raw Probabilities Only — No Labels in API Responses

All prediction endpoints return **raw P(Late) probabilities**. They do NOT return labels like "BERISIKO" or "LAYAK". The **frontend** applies the threshold and determines the label client-side. This means:

- Moving the threshold slider → instant label update → **zero API calls**
- The gauge needle, risk label, recommendation text, and confusion matrix all update in the browser
- The entire threshold sweep on the test-set analysis page is client-side

### 2. Self-Contained Deployment

All ML code (`src/`), model artifacts (`models/`), processed data (`data/processed/`), and figures (`docs/figures/`) are included directly in this project. This ensures:

- No runtime dependency on the parent `debt_prediction/` project
- Easy deployment to any environment (Docker, cloud, etc.)
- Everything needed to run the API is in one directory

### 3. Separate Virtual Environment

This project uses its own `pyproject.toml` and `.venv/` so it can:

- Have its own dependency versions (e.g., FastAPI, uvicorn) without conflicting with the parent project's Streamlit dependencies
- Be tracked in a separate git repository
- Be deployed independently

### 4. Graceful Degradation

If optional artifacts (MLP, SHAP, calibration data) are missing, the API still starts and serves what it can:

- `/api/health` reports `"degraded"` status with details on what's missing
- `/api/predict` returns XGBoost predictions even if MLP is unavailable
- Analysis endpoints return `null` for missing optional data
